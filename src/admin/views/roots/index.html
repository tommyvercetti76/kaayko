∂ Scientific Foundation & Research Basis▿
Scoring model · Domain research basis · Integrity framework · Patent-defensible methodology
Scoring Engine
Bayesian Beta-Conjugate with Polytomous Graded Response
Each domain score is a Beta distribution posterior: Beta(1 + Σα_updates, 1 + Σβ_updates). Score = E[θ] × 100. Confidence = 1 − 2σ (posterior concentration). Prior: Beta(1,1) — uninformative uniform prior.
Raiffa & Schlaifer (1961) · Gelman et al. (2013) · Samejima (1969) Graded Response Model · Lord (1980) IRT
Authenticity Index (PRAI)
Pareekshā Response Authenticity Index
Three signals from Arthaśāstra integrity framework: Dharmopadhā (streak bias), Arthopadhā (CV uniformity), Bhayopadhā (timing uniformity). PRAI = 1 − w₁·S − w₂·CV − w₃·T. Flags but does not modify domain scores.
Kautilya (Arthaśāstra 1.10) · Paulhus (1984) impression management · Lindley (1956) sequential Bayesian
Pancha-Kosha Domain Architecture
Taittirīya Upaniṣad 2.1–5 — 5 layers: Annamaya (Physical), Pranamaya (Resilience/Music), Manomaya (Empathy/Character), Vijnanamaya (Curiosity/Systems), Anandamaya (Spiritual). Cross-domain propagation by Kosha distance: same-layer r=0.55, adjacent r=0.35, distant r=0.20.
Taittirīya Upaniṣad (800–600 BCE) · Combs (2002) integral psychology · Bronfenbrenner ecological systems

Multi-Axis Polytomous Answer Mapping
PATENT CLAIM
Each answer option simultaneously updates multiple domain posteriors — a single response propagates evidence across the Pancha-Kosha axis space based on per-option axis weights authored into the question bank. Standard IRT assumes one item → one dimension; this system implements a structured polytomous multi-axis update where axis contribution weights are independently determined per answer option.
// Example: Parent resilience question, Answer option A
answer.axes = {
resilience: 5,   // primary axis — "models persistence explicitly"
empathy:   3,   // secondary — "acknowledges child feelings"
character: 2   // tertiary — "sets behavioral expectation"
}
// CAT engine update for each axis (conjugate Beta update):
posterior['resilience'].alpha += axes.resilience / 5  → α += 1.00
posterior['empathy'].alpha   += axes.empathy / 5    → α += 0.60
posterior['character'].alpha += axes.character / 5   → α += 0.40
// β updates = (1 − normalized score) for each axis independently
Information Efficiency
One question propagates evidence across up to 3 domains simultaneously — effectively 3× the information density of standard unidimensional IRT per question-slot
Structured Authoring
Axis weights are independently validated per answer option in the question bank — not inferred statistically. Authors assign weights based on construct overlap, reviewed by psychometric committee
Patent Distinction
Distinct from MIRT (Multidimensional IRT): MIRT infers cross-domain loadings from data; this system uses content-validated expert-assigned axis weights embedded in the question-answer structure
Samejima (1969) graded response model · Reckase (2009) multidimensional IRT · Embretson & Reise (2000) IRT for psychologists · Implementation: cat-engine.service.ts updatePosterior()
Closed-Loop IRT Calibration (Claim 5)
PATENT CLAIM 5
The system implements a self-calibrating IRT pipeline that automatically upgrades question difficulty parameters as empirical response data accumulates. Item parameters improve across 4 tiers without human intervention, using real Pearson item-total correlations (rit) computed from session response history.
Tier 0 — Uncalibrated
N < 50
Default b=0, a=0.577
(rit=0.5 bootstrap)
Tier 1 — Rasch (1PL)
N ≥ 50
b = logit(fail%)
difficulty only
Tier 2 — 2PL
N ≥ 200
a = rit / √(1−rit²)
Real rit from sessions
Tier 3 — 3PL Flag
N ≥ 500
Guessing parameter c
→ manual MLE review
Live rit pipeline (active as of 2026-03-29): On each session completion, per-question response records (gradeScore, item-excluded domainTotalScore) are written to roots_question_responses/{questionId}/responses. At calibration time, rit = Pearson(itemScores, domainTotals) is computed directly from this collection (min N=30 for statistical validity; falls back to 0.5 bootstrap below threshold). Lord's approximation then derives the 2PL discrimination parameter a. All calibration events are logged to roots_learning_reports.
Lord (1980) IRT · Lord's approximation a = rit/√(1−rit²) · Nunnally (1978) minimum rit=0.20 · Implementation: irt-calibration.service.ts + prototype-integration.ts onSessionCompleted()

Ψ PRAI Mathematical Reference (Theoretical)▹
PRAI = 1 − (0.40·streakP + 0.35·cvP + 0.25·timeP) · Session quality metadata — does NOT modify domain scores · Streak threshold = 6
Streak Component Effect on PRAI (CV=0, Time=0)
Shows streak signal only (w=0.40) · threshold=6 · Actual PRAI is lower when CV and timing penalties are non-zero
Beta(1,1): Score Range After N Questions
All-S vs All-F posterior mean — theoretical ceiling & floor · Scores are independent of PRAI
ⓘ Architecture Note: PRAI is a session quality indicator (anumāna / inference). It does not multiply or alter domain scores (pratyākṣa / direct measurement). Domain scores reflect what the Bayesian engine measured; PRAI flags whether the evidence is trustworthy. Admissions counsellors see PRAI as metadata for informed decisions.

▶ Session Simulator — Generate Realistic Test Data▿
GRM-based synthetic sessions · Varied grades per domain · Full scoring pipeline · Instantly visible in Sessions tab
How it works: Each session gets independent per-domain theta values drawn from the selected profile. Grades are sampled via the Graded Response Model (a=4) — so a parent with θ=0.3 on emotional availability will answer those questions with C or D, while scoring B/A on their stronger domains. The full scoring, PRAI, and report pipeline runs on each session. Sessions appear immediately in the Sessions tab tagged isSimulated:true.
