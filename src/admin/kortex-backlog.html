The vendor journey dies at step 2. A zoo, sports club, or conference org clicks "Get Started Free" and gets a success message promising an email that never arrives. No tenant is created, no auth user exists, no login is possible. Everything behind the login wall (dashboard, analytics, campaigns, QR codes) works — but nobody can get there.

What Works (behind the login wall)

• Admin SPA with 8 views (dashboard, create, all links, analytics, campaigns, QR, billing, onboarding)
• Smart link CRUD + platform-aware redirect + A/B routing
• Campaign management with lifecycle (active/paused/archived)
• Click analytics with device/browser/UTM/referrer breakdown
• QR code generation + download
• Multi-tenant architecture with role-based access
• 6-layer security (HMAC, velocity profiling, honeypots, bot detection)
• Public API + HMAC-signed webhooks + batch operations

What's Broken (self-serve blockers)

• Signup → pending review, not auto-provisioned tenant + auth user
• No email system — no welcome email, no alerts, no reports
• Onboarding wizard — all 6 steps save to local JS state, zero API calls
• Stripe placeholder IDs — "price_pro_monthly" not real Stripe products
• External URLs blocked — tenant admins can't point links to their own domains
• No guided empty state — new tenants see blank dashboard with no guidance
• Weekly digest logic exists but no scheduler wired, no email sender
• Landing page says "Universities" — no venue/sports/entertainment use cases
